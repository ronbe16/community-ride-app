## Project Overview
Community Ride is a mobile-first PWA for HOA (homeowners association) community members to coordinate carpooling. Drivers post trips, passengers join, and the app facilitates safety exchanges (photo verification) and manifests. It is built as a Vite + React SPA with Firebase as the backend (Firestore + Auth + FCM). The app is currently in active beta (v1.0.4), deployed at `community-ride.lovable.app`.

## Known False Positives
- All `VITE_FIREBASE_*` variables appear to be client-side secrets but this is **intentional and expected** — Firebase web API keys are designed to be public. Security is enforced by Firestore Security Rules, not key secrecy.
- `VITE_CLOUDINARY_UPLOAD_PRESET` is an unsigned upload preset — it is intentionally public. Cloudinary security is enforced by upload preset restrictions in the Cloudinary dashboard.
- `src/components/ui/` files are auto-generated shadcn/ui components. Do not flag shadcn internals as issues.
- **Gitleaks historical .env findings** — commits `e3556f3` and `102d8b3` contain `.env` with Firebase API key and VAPID key in git history. Pre-existing, already known (tracked as TOOLS-001 since Pass 1), not actionable via code changes. Do not re-flag these as new findings.
- **BIZ-007 / Manifest `expiresAt` not enforced as a security gate** — informational by design. LTFRB compliance requires the passenger manifest to remain publicly accessible for inspection after the trip concludes, not just during the trip window. The `expiresAt` field is a display hint only; Firestore document lifetime (`deleteAt`) controls actual data retention. Do not flag Manifest.tsx for missing expiry enforcement.
- **OPS-001 / ErrorBoundary not wired to Sentry** — deferred, not a priority for the current HOA alpha stage. The ErrorBoundary falls back gracefully and console.error is sufficient for now. Do not re-flag this in future audit passes until explicitly re-opened.

## Key Architectural Decisions
- **Firebase instead of Supabase** — this app uses Firebase Firestore + Firebase Auth, not Supabase. There are no SQL migrations, no RLS policies, and no edge functions. Security is enforced by Firestore Security Rules (not included in this repo — verify in Firebase Console).
- **All env vars are VITE_ prefixed** — this is intentional. There is no server-side backend in this repo; all Firebase calls are made from the browser using the Firebase web SDK with security enforced by Firestore rules.
- **Cloudinary for images** — Firebase Storage is initialized but not used for uploads. All image uploads go to Cloudinary with unsigned presets.
- **Consent versioning** — `CONSENT_VERSION` in `src/constants/app.ts` drives the re-consent modal. Bumping it forces all existing users to re-accept on next login.
- **90-day retention** — users and trips have a `deleteAt` field. Cleanup is expected to run via a scheduled Cloud Function (not in this repo).
- **No subscription/tier logic** — this is a community app with no paid tiers. Skip all subscription/tier audit checks.
- **No Supabase** — skip all Supabase-specific audit checks (RLS, edge functions, migrations).
- **PWA** — the app is installable as a PWA; FCM push notifications require HTTPS.
